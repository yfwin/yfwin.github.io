Github 网站 DNS 被污染解决方法

发表于 2021-12-17 本文字数 920 字 阅读时长 1 分钟
  • 逸风小站
首页 Github

报错 curl: (7) Failed to connect to raw.githubusercontent.com port 443: Connection refused

# 解决方法：

参考：
https://bbs.huaweicloud.com/blogs/143682
很多方法都没用，这个解决了问题。希望对你有用。

原因：由于某些你懂的因素，导致 GitHub 的 raw.githubusercontent.com 域名解析被污染了。

解决方法：通过修改 hosts 解决此问题。

# 查询真实 IP

在 https://www.ipaddress.com/ 查询 raw.githubusercontent.com 的真实 IP。

修改 hosts

sudo vim /etc/hosts

添加如下内容：

199.232.68.133 raw.githubusercontent.com

重新执行命令即可，但是下载会比较慢。

最后提一嘴，电脑连上你的手机热点有可能会有奇效。

原文链接：https://blog.csdn.net/sscc_learning/article/details/105574354

# github.com 无法访问

也是在 https://www.ipaddress.com/ 查询 github.com 的真实 IP。

# mac 和 windows 系统的 hosts 位置

mac 系统:/etc/hosts
win: C:\Windows\System32\drivers\etc\HOSTS
windows 系统设置完要刷新一下 dns，

ipconfig/flushdns

# Hosts 自动刷新工具 SwitchHosts

SwitchHosts 是一个管理、切换多个 hosts 方案的工具。
它是一个免费开源软件。

网站：https://swh.app/zh/

https://raw.hellogithub.com/hosts
